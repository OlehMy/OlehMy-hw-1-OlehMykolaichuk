Перелік основних команд в Git

  • git config --global user.name "Your Name" - встановлює ім'я, яке буде відображатися в полі автора у комітах
  • config --global user.email your@email - встановлює адресу електронної пошти, яка буде відображатися в інформації про коміти
  • git init - створює новий локальний репозиторій з заданим ім'ям
  • git status - показує стан файлів в робочій директорії і індексі: які файли змінені, але не додані в індекс; які очікують коміту в індексі. Додатково також виводить підказки про те, як змінити стан файлів
  • git add - додає файли робочої дерикторії в їндекс для наступного коміту. Використовуємо " . " або " * " для додавання усіх файлів. Щоб додати один файл – вказуємо його ім'я та розширення
  • git commit -м "Повідомлення з описом коміту" - фіксує проіндексовані зміни і зберігає їх в історію версій. Можна вказати без аргументу -m, в такому випадку відкриється текстовий редактор по замовчуванню, в якому потрібно буде описати коміт. Якщо це сталось випадково, вводимо :q і натискаємо Enter
  • git log - показує історію комітів, починаючи з самого останнього. За замовчуванням показує лише історію поточної гілки, але може бути налаштована на виведення історії інших гілок (навіть кількох одночасно). Також її можна використовувати для перегляду відмінностей між гілками на рівні комітів
  • git checkout - використовується для переходу між гілками та перенесення їх вмісту в робочу дерикторію
  • git branch - використовується для керування гілками в репозиторії
  • git push origin master - відправляє в репозиторій зліпки проекту. origin – назва віддаленого репозиторію (зазвичай вибирають origin, але в теорії може бути будь-яке слово). master – назва гілки в яку додаємо зміни (master – головна гілка у репозиторії, може бути будь якою)
  • git pull origin master – команда для отримання змін з репозиторію. origin – назва віддаленого репозиторію (зазвичай вибирають origin, але в теорії може бути будь яке слово). master – назва гілки в яку додаємо зміни (master – головна гілка у репозиторії, може бути будь-якою)
  • git merge - використовується для злиття однієї або декількох гілок в поточну зі збереженням історії змін вихідних гілок
  • git clone - використовується для копійування існуючого віддаленого репозиторію у вказану папку на комп'ютері
  • git revert - створює новий коміт, який скасовує зміни, внесені в зазначений коміт, і застосовує його до поточної гілки
  • git reset - скасовує всі коміти після заданого, залишаючи всі зміни в робочій директорії
  • git rebase - використовується для перенесення змін з однієї гілки в іншу з перезаписом в історії змін. rebase зжимає усі зміни в один патч, і інтегрує його в цільову гілку
  • git cherry-pick - використовується для того, щоб взяти зміни, внесені будь яким комітом, і спробувати застосувати їх заново у вигляді нового коміту на верху поточної гілки
  • git stash - зберігає в сховищі незавершених змін змінені відслідковувані файли та проіндексовані зміни (наприклад при переході на іншу гілку без потреби їх фіксації) з подальшою можливістю повернутися до роботи з файлом
  • . gitignore - це файл, який потрібен для приховування файлів і папок (конфігураційні файли, тимчасові файли і папки) від системи контролю версій Git
